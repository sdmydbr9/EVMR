EVMR: Electronic Veterinary Medical Records System

A comprehensive microservice-based application for managing veterinary practices, medical records, appointments, inventory, and clinic operations.

System Overview

EVMR is designed to streamline veterinary practice management with the following modules:

Patient Management

Store and retrieve complete patient records with comprehensive medical history.

Electronic Medical Records

SOAP format medical documentation with support for attachments and lab results.

Appointment Scheduling

Schedule and manage appointments with automated reminders and confirmations.

Inventory Management

Track medications and supplies with automatic reordering and expiration alerts.

Reporting & Analytics

Generate insights from practice data with customizable reports and dashboards.

Multi-Location Management

Support for multiple clinics and staff roles with centralized administration.

Architecture

EVMR follows a microservice architecture pattern, with separate services for authentication, patient management, appointments, inventory, and more. This architecture provides scalability, resilience, and flexibility.

Tech Stack

  • Frontend: React.js with Material UI
  • Backend: Node.js with Express
  • Databases: MongoDB (microservices) and PostgreSQL (legacy system)
  • Authentication: JWT-based authentication with Redis caching
  • Containerization: Docker and Docker Compose

Getting Started

To get started with EVMR, follow these steps: 

# Clone the repository
git clone <repository-url>
cd EVMR

# Set up environment variables
cp .env.example .env

# Start the application with Docker Compose
docker-compose up -d

For more detailed instructions, see the Quick Start Guide.
