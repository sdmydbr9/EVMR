Patient Service

The Patient Service is the core of the EVMR system, responsible for managing patient records, medical history, and related information.

On this page

  • Overview
  • Architecture
  • Key Features
  • API Endpoints
  • Data Model
  • Configuration

  • Deployment

Overview

The Patient Service manages all patient-related data, including personal information, medical history, vaccinations, allergies, and medications. It provides a comprehensive API for creating, retrieving, updating, and deleting patient records.

Service Details

  • Port: 5102
  • Base URL: http://patient-service:5102
  • Technologies: Node.js, Express, MongoDB
  • Dependencies: Auth Service (for authentication)

Architecture

The Patient Service follows a layered architecture pattern:

Figure 1: Patient Service internal architecture

Components

  • API Layer: Express.js routes and controllers
  • Service Layer: Business logic for patient management
  • Data Access Layer: MongoDB models and repositories
  • Auth Middleware: JWT validation and authorization

API Endpoints

The Patient Service exposes the following RESTful API endpoints:

Patient Management

GET
/api/patients
List all patients (with pagination and filtering)
GET
/api/patients/:id
Get patient details by ID
POST
/api/patients
Create a new patient record
PUT
/api/patients/:id
Update patient information
DELETE
/api/patients/:id
Delete patient (soft delete)
GET
/api/patients/search
Search patients by name, species, breed, etc.

Medical Records

/api/medical-records/patient/:patientId
Get all medical records for a patient
GET
/api/medical-records/:id
Get medical record by ID
POST
/api/medical-records
Create a new medical record
PUT
/api/medical-records/:id
Update medical record
DELETE
/api/medical-records/:id
Delete medical record

Vaccinations

POST
/api/patients/:id/vaccinations
Add vaccination record
GET
/api/patients/:id/vaccinations
Get vaccination history
PUT
/api/patients/:patientId/vaccinations/:id
Update vaccination record

Configuration

The Patient Service is configured using environment variables: 

# Server Configuration
PORT=5102
NODE_ENV=development

# MongoDB Configuration
MONGODB_URI=mongodb://mongodb:27017/patient-service

# Auth Service Configuration
AUTH_SERVICE_URL=http://auth-service:5101

# Logging
LOG_LEVEL=info

Integration with Other Services

The Patient Service integrates with other microservices in the following ways:

Auth Service Integration

The Patient Service relies on the Auth Service for authentication and authorization. It validates JWT tokens and retrieves user information. 

// Example authentication middleware
const authenticateJWT = async (req, res, next) => {
  try {
    const token = req.header('Authorization')?.replace('Bearer ', '');
    
    if (!token) {
      return res.status(401).json({ error: 'Authentication required' });
    }
    
    // Validate token with Auth Service
    const response = await axios.get(`${process.env.AUTH_SERVICE_URL}/api/auth/verify`, {
      headers: { Authorization: `Bearer ${token}` }
    });
    
    req.user = response.data.user;
    next();
  } catch (error) {
    res.status(401).json({ error: 'Invalid token' });
  }
};

Appointment Service Integration

The Appointment Service retrieves patient information from the Patient Service when scheduling appointments.

Deployment

The Patient Service is deployed as a Docker container as part of the EVMR microservices architecture.

Docker Configuration

# From docker-compose.yml
patient-service:
  build:
    context: ./services/patient-service
    dockerfile: Dockerfile
  container_name: petsphere-patient-service
  restart: unless-stopped
  ports:
    - "5102:5102"
  depends_on:
    mongodb:
      condition: service_healthy
    auth-service:
      condition: service_healthy
  environment:
    - NODE_ENV=${NODE_ENV}
    - PORT=5102
    - MONGODB_URI=mongodb://mongodb:27017/patient-service
    - AUTH_SERVICE_URL=http://auth-service:5101
    - LOG_LEVEL=info
  env_file:
    - .env
  networks:
    - petsphere-network
  healthcheck:
    test: ["CMD", "wget", "-qO-", "http://localhost:5102/health"]
    interval: 30s
    timeout: 5s
    retries: 3
    start_period: 30s
