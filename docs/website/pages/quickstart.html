Quick Start Guide

Get up and running with EVMR in minutes. This guide will help you set up a development environment and explore the system's core features.

Prerequisites

Before you begin, ensure you have the following installed on your system:

  • Docker (version 20.10.0 or higher)
  • Docker Compose (version 2.0.0 or higher)
  • Git (version 2.30.0 or higher)
  • Node.js (version 16.0.0 or higher) - for development only
  • npm (version 8.0.0 or higher) - for development only

System Requirements

Recommended system specifications for running EVMR:

  • CPU: 4 cores or more
  • RAM: 8GB or more
  • Disk Space: 10GB free space
  • Operating System: Linux, macOS, or Windows with WSL2

Installation

Follow these steps to install EVMR:

1. Clone the Repository

git clone https://github.com/yourusername/EVMR.git
cd EVMR

2. Set Up Environment Variables

Create a .env file in the root directory by copying the example file: 

cp .env.example .env

Open the .env file and update the variables as needed. The most important variables to configure are: 

# Database Configuration
MONGODB_URI=mongodb://mongodb:27017/evmr
POSTGRES_URI=postgresql://postgres:postgres@postgres:5432/evmr_legacy

# JWT Secret
JWT_SECRET=your_jwt_secret_key
JWT_EXPIRATION=1d
JWT_REFRESH_EXPIRATION=7d

# API Configuration
API_PORT=5000
NODE_ENV=development

# Email Configuration (for notifications)
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=your_email@example.com
SMTP_PASS=your_email_password

Configuration

EVMR uses Docker Compose for container orchestration. The default configuration should work for most development environments, but you can customize it by editing the docker-compose.yml file.

Default Ports

The following ports are used by default:

Service Port Description
Frontend 3000 React application
API Gateway 5000 Main API entry point
Auth Service 5101 Authentication service
Patient Service 5102 Patient management service
Appointment Service 5103 Appointment scheduling service
MongoDB 27017 MongoDB database (internal)
PostgreSQL 5432 PostgreSQL database (internal)
Redis 6379 Redis cache (internal)

Running the Application

Start the application using Docker Compose: 

docker-compose up -d

This command will:

  1. Build or pull all required Docker images
  2. Create and start containers for all services
  3. Set up the network between containers
  4. Initialize the databases with sample data

First-Time Startup

The first time you start the application, it may take several minutes to download images and initialize the databases. Subsequent startups will be much faster.

Checking Service Status

Verify that all services are running: 

docker-compose ps

You should see all services with a status of "Up".

Viewing Logs

To view logs from all services: 

docker-compose logs -f

To view logs from a specific service: 

docker-compose logs -f [service-name]

Accessing the Application

Once all services are running, you can access the application:

  • Frontend: http://localhost:3000
  • API Gateway: http://localhost:5000
  • API Documentation: http://localhost:5000/api-docs

Stopping the Application

To stop the application: 

docker-compose down

To stop the application and remove all data (including databases): 

docker-compose down -v

Warning

The -v flag will remove all volumes, including database data. Use with caution in production environments.
