Contributing to EVMR

Thank you for your interest in contributing to the EVMR project! This guide will help you get started with contributing to both the codebase and documentation.

Code of Conduct

The EVMR project is committed to fostering an open and welcoming environment. By participating, you are expected to uphold our Code of Conduct, which includes:

  • Using welcoming and inclusive language
  • Being respectful of differing viewpoints and experiences
  • Gracefully accepting constructive criticism
  • Focusing on what is best for the community
  • Showing empathy towards other community members

Getting Started

To get started with contributing to EVMR, follow these steps:

  1. Fork the Repository

    Visit the EVMR GitHub repository and click the "Fork" button in the top-right corner.

  2. Clone Your Fork

    git clone https://github.com/your-username/EVMR.git
cd EVMR

  3. Add the Upstream Remote

    git remote add upstream https://github.com/yourusername/EVMR.git

  4. Create a Branch

    git checkout -b feature/your-feature-name

Development Workflow

The EVMR project follows a feature branch workflow:

  1. Create a new branch for each feature or bugfix
  2. Make your changes in the branch
  3. Write tests for your changes
  4. Run the test suite to ensure all tests pass
  5. Submit a pull request

To keep your fork up to date with the main repository: 

git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Pull Requests

When submitting a pull request, please:

  • Provide a clear and descriptive title
  • Reference any related issues using the GitHub issue number (e.g., "Fixes #123")
  • Include a detailed description of the changes
  • Update documentation if necessary
  • Add tests for new features or bug fixes
  • Ensure all tests pass

Pull request template: 

## Description
[Provide a brief description of the changes]

## Related Issues
[Reference any related issues, e.g., "Fixes #123"]

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation update
- [ ] Performance improvement
- [ ] Code refactoring
- [ ] Other (please describe)

## Checklist
- [ ] My code follows the project's coding standards
- [ ] I have added tests for my changes
- [ ] All tests pass locally
- [ ] I have updated the documentation accordingly
- [ ] My changes don't introduce new warnings or errors

Coding Standards

The EVMR project follows these coding standards:

  • JavaScript/Node.js: ESLint with Airbnb style guide
  • React: ESLint with React-specific rules
  • CSS: Stylelint with standard configuration
  • Commit Messages: Conventional Commits format

To check your code against our standards: 

# Install dependencies
npm install

# Run linting
npm run lint

# Run linting with auto-fix
npm run lint:fix

Documentation

Documentation is a crucial part of the EVMR project. We welcome contributions to improve our documentation:

  • Fix typos or clarify existing documentation
  • Add examples or tutorials
  • Create diagrams or visual aids
  • Translate documentation to other languages

The documentation is located in the docs/ directory and is built using HTML, CSS, and JavaScript. To contribute to the documentation:

  1. Make your changes to the relevant files
  2. Preview the changes locally by opening the HTML files in your browser
  3. Submit a pull request with your changes

Testing

The EVMR project uses the following testing frameworks:

  • Backend: Jest for unit and integration tests
  • Frontend: React Testing Library and Jest
  • End-to-End: Cypress

To run the tests: 

# Run all tests
npm test

# Run backend tests
npm run test:backend

# Run frontend tests
npm run test:frontend

# Run end-to-end tests
npm run test:e2e

When contributing new features or fixing bugs, please include tests that cover your changes.

Issue Reporting

If you find a bug or have a feature request, please create an issue on the GitHub issue tracker.

When reporting a bug, please include:

  • A clear and descriptive title
  • Steps to reproduce the issue
  • Expected behavior
  • Actual behavior
  • Screenshots or error messages (if applicable)
  • Environment information (browser, OS, etc.)
